# Task 009: RFP 업로드 페이지 UI 구현

## 상태
- [ ] 미완료

## 개요
RFP 업로드 페이지의 레이아웃을 구현합니다. RFP 이력 목록 테이블, 파일 업로드 영역(드래그앤드롭),
AI 파싱 진행 상태 표시, 파싱 결과 표시 및 수동 수정 기능을 포함합니다.

## 관련 PRD 기능
- [F003] RFP 업로드
- [F004] RFP AI 파싱

## 관련 파일
- `app/(auth)/rfp/page.tsx` — RFP 업로드 페이지
- `components/rfp/rfp-table.tsx` — RFP 이력 목록 테이블
- `components/rfp/rfp-upload-zone.tsx` — 파일 업로드 영역
- `components/rfp/parsing-progress.tsx` — AI 파싱 진행 상태
- `components/rfp/parsing-result.tsx` — 파싱 결과 표시 및 수정

## 수락 기준
- [ ] RFP 이력 목록 테이블이 더미 데이터로 표시됨
- [ ] 파일 업로드 영역이 드래그앤드롭을 지원함
- [ ] AI 파싱 진행 상태 프로그레스 바가 표시됨
- [ ] 파싱 결과 표시 및 수동 수정/보완 인라인 편집이 동작함
- [ ] 견적 생성하기 버튼이 표시됨

## 구현 단계
- [ ] RFP 이력 목록 테이블 구현 (날짜, 파일명, 파싱 상태, 연결된 견적 수)
- [ ] 파일 업로드 영역 구현 (드래그앤드롭, PDF/HWP/DOCX 지원 표시)
- [ ] AI 파싱 진행 상태 프로그레스 바 구현
- [ ] 파싱 결과 표시 영역 구현
- [ ] 추출된 요구사항 목록 (서버 대수, CPU, 메모리, 스토리지, 네트워크 등)
- [ ] 수동 수정/보완 인라인 편집 기능
- [ ] RFP 문서 삭제 기능 UI 구현
- [ ] 기존 RFP 선택 시 파싱 결과 재표시 구현
- [ ] 견적 생성하기 버튼 구현

## 테스트 체크리스트
- [ ] 파일 드래그앤드롭 동작 확인
- [ ] RFP 이력 테이블 더미 데이터 표시
- [ ] 파싱 결과 인라인 편집 동작
- [ ] 견적 생성하기 버튼 클릭 시 페이지 이동

## 참고사항
더미 데이터 기반으로 UI만 구현하며, 실제 파일 업로드 및 AI 파싱은 Task 016에서 구현합니다.
Task 005의 파일 업로드 컴포넌트와 데이터 테이블 컴포넌트를 재사용합니다.
