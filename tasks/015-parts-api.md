# Task 015: 부품 DB 관리 API 및 기능 연동 (F001, F002)

## 상태
- [ ] 미완료

## 개요
부품 CRUD API와 부품 가격 관리 API, 카테고리 관리 API를 구현합니다.
부품 관리 페이지의 더미 데이터를 실제 API 호출로 교체하고, 역할 기반 접근 제어를 적용합니다.

## 관련 PRD 기능
- [F001] 부품 관리
- [F002] 부품 가격 관리

## 관련 파일
- `app/api/parts/route.ts` — 부품 목록 조회 / 등록 API
- `app/api/parts/[id]/route.ts` — 부품 수정 / 삭제 API
- `app/api/categories/route.ts` — 카테고리 추가 API
- `app/api/categories/[id]/route.ts` — 카테고리 삭제 API
- `app/(auth)/parts/page.tsx` — 부품 관리 페이지 (API 연동)

## 수락 기준
- [ ] 부품 CRUD API가 정상 동작함 (등록, 조회, 수정, 삭제)
- [ ] 원가 암호화 저장/복호화 조회가 동작함
- [ ] 마진 자동 계산 로직이 동작함
- [ ] 카테고리 관리 API가 동작함
- [ ] 부품 관리 페이지가 실제 API와 연동됨
- [ ] 역할 기반 접근 제어가 적용됨 (관리자: CRUD, 멤버: 조회만)

## 구현 단계
- [ ] `POST /api/parts` 부품 등록 API 구현
- [ ] `GET /api/parts` 부품 목록 조회 API 구현 (카테고리 필터, 검색, 페이지네이션)
- [ ] `PUT /api/parts/[id]` 부품 수정 API 구현
- [ ] `DELETE /api/parts/[id]` 부품 삭제 API 구현
- [ ] 원가 암호화 저장/복호화 조회 처리 구현
- [ ] 마진 자동 계산 로직 구현
- [ ] `POST /api/categories` 카테고리 추가 API 구현
- [ ] `DELETE /api/categories/[id]` 카테고리 삭제 API 구현 (하위 부품 0건 검증)
- [ ] 부품 관리 페이지 더미 데이터를 실제 API 호출로 교체
- [ ] 역할 기반 접근 제어 적용 (관리자 이상: CRUD, 멤버: 조회만)
- [ ] API 통합 테스트

## 테스트 체크리스트
- [ ] 부품 등록/조회/수정/삭제 API 동작 확인
- [ ] 원가 암호화/복호화 정확성 확인
- [ ] 카테고리 추가/삭제 동작 확인
- [ ] 하위 부품 존재 시 카테고리 삭제 차단 확인
- [ ] 멤버 역할로 수정/삭제 시도 시 403 응답
