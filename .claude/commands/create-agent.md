# /create-agent - 에이전트 생성

목적에 맞는 새로운 전문 에이전트를 리서치하고 생성합니다. 이 커맨드는 `agent-creator` 에이전트를 호출하여 웹에서 모범사례를 조사하고, 기존 팀 구조를 분석한 뒤 최적의 에이전트를 설계/생성합니다.

> **팀장이 직원을 뽑듯이**: 필요한 역할을 설명하면, 해당 분야의 베스트 프랙티스를 조사하여 최적의 에이전트를 만들고 팀에 합류시킵니다.

## 입력

생성할 에이전트의 역할 설명: $ARGUMENTS

## 생성 절차

### 1단계: 요구 분석

사용자 요청에서 역할, 전문 분야, 기대 산출물을 추출합니다.

기존 에이전트 목록(`.claude/agents/*.md`)을 확인하여 역할 중복 여부를 검증합니다:
- 중복 발견 시 기존 에이전트 확장을 먼저 제안
- 중복 없을 시 새 에이전트 설계 진행

### 2단계: 웹 리서치

WebSearch로 해당 분야의 에이전트 설계 패턴을 조사합니다 (2-3회):
- 해당 분야의 프롬프트 엔지니어링 모범사례
- 필요한 도구, 워크플로우, 출력 형식

### 3단계: 설계안 제시

에이전트 이름, 역할, 핵심 기능, 도구, 산출물을 구조화하여 사용자에게 제시합니다.

사용자 승인을 받은 후에만 파일을 생성합니다.

### 4단계: 파일 생성

승인 후 다음 파일을 생성/수정합니다:
- `.claude/agents/{name}.md` — 에이전트 정의
- `.claude/commands/{command}.md` — 슬래시 커맨드
- `.claude/agents/leader.md` — 서브 에이전트 테이블에 추가
- `CMDHelp.md` — 커맨드 도움말에 추가

### 5단계: 완료 보고

생성된 파일 목록, 사용법, 호출 예시를 안내합니다.

## 예시

- `/create-agent 프론트엔드 성능 최적화 전문가`
- `/create-agent DB 마이그레이션 관리자`
- `/create-agent API 테스트 자동화 전문가`
- `/create-agent 접근성(a11y) 검수 전문가`
- `/create-agent SEO 최적화 에이전트`

## 주의사항

- 에이전트 생성 전 반드시 사용자 승인을 받습니다
- 기존 에이전트와 역할이 중복되면 확장을 먼저 제안합니다
- 생성된 에이전트는 Leader에 자동 등록되어 즉시 활용 가능합니다
