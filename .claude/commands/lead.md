# /lead - 리더 에이전트 호출

리더 에이전트를 통해 서브 에이전트 팀을 조율합니다. 사용자 지시를 분석하여 적절한 워크플로우 모드를 자동 선택하고, 서브 에이전트에 업무를 위임한 후 결과를 취합합니다.

## 입력

지시사항: $ARGUMENTS

## 실행 절차

### 1단계: 지시 분석

사용자 지시를 분석하여 워크플로우 모드를 결정합니다:

- **"처음부터", "새 프로젝트", "전체 기획"** → 모드 1: 전체 기획 (5 Stage)
- **"기능 구현", "개발 계획"** → 모드 2: 기능 개발 (1-2 Stage)
- **특정 업무 요청** → 모드 3: 단일 업무 위임

### 2단계: 모드 안내

선택된 모드와 실행할 Stage를 사용자에게 안내합니다:

```
🎯 모드: [전체 기획 / 기능 개발 / 단일 업무]
📋 실행 계획:
  Stage 1: [설명] — [담당 에이전트]
  Stage 2: [설명] — [담당 에이전트]
  ...

진행할까요?
```

### 3단계: Stage 순차 실행

각 Stage를 순서대로 실행합니다:

1. 서브 에이전트에 업무 위임 (이전 Stage 결과를 컨텍스트로 전달)
2. 서브 에이전트 결과 수신
3. Gate Review 출력 (결과 요약, 산출물, 결정사항, 다음 단계)
4. 사용자 승인 대기
5. 승인 시 다음 Stage로 진행

### 4단계: 최종 리포트

모든 Stage 완료 후 최종 리포트를 작성합니다.

## 서브 에이전트 매핑

| 키워드 | 에이전트 | 역할 |
|--------|---------|------|
| 시장/경쟁/SWOT | service-analyst | 시장 및 서비스 분석 |
| PRD/요구사항 | prd-generator | PRD 작성 |
| 검증/기술 검토 | prd-validator | PRD 기술 검증 |
| 아키텍처/기술 결정 | architect | 아키텍처 결정 |
| 로드맵/일정 | development-planner | ROADMAP.md 생성 |
| 구현 계획/기능 기획 | planner | 기능 구현 계획 |

## 사용 예시

```bash
# 전체 기획
/lead 온라인 중고 거래 플랫폼을 처음부터 기획해줘

# 기능 개발
/lead 사용자 검색 기능을 구현할 계획을 세워줘

# 단일 업무
/lead 현재 PRD를 기술적으로 검증해줘
```

## 주의사항

- 리더 에이전트는 직접 코드를 작성하지 않습니다
- 각 Stage 완료 후 반드시 사용자 승인을 받습니다
- `docs/planning/progress.md`를 자동으로 업데이트합니다
- Slack MCP가 연결되어 있으면 Stage 완료 시 알림을 전송합니다
